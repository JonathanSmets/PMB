Notices UNIMARC

L'import de notices UNIMARC[1] permet d'intégrer dans le catalogue des notices sans exemplaires associés.

 

Le fichier à importer peut être transmis de deux façons :

  • par téléchargement depuis l'interface du logiciel en cliquant sur Téléchargement de votre fichier.

  • en déposant le fichier sur le serveur dans le répertoire .../pmb/admin/import (avec un client FTP par exemple), puis en cliquant Pré-chargement du fichier.

    Ceci permet de transférer des fichiers sans tenir compte de la taille maximum de téléchargement imposée par PHP[2] .

    (Cette limite est fixée par la configuration du serveur, dans le fichier php.ini).

    Il faudra faire attention à renommer le fichier conformément au nom précisé dans l'interface.

 

Il est également possible de choisir la fonction d'import à utiliser, ainsi que l'encodage de la source.

Truc & astuce

Si un fichier UNIMARC[1] ne peut être importé, vous pouvez en vérifier la validité en réalisant une conversion de ce fichier vers un format lisible.

 

Pour plus d'informations, voir le chapitre : Conversions/export

Transfert du fichier par téléchargement⚓

Les options suivantes sont possibles lors de l'import :

  • ISBN obligatoire ? :

    Permet de définir si l'ISBN doit être toujours présent dans les notices importées (ce n'est pas toujours le cas, attention aux CD audio, cédéroms et DVD ainsi qu'à la littérature grise ou non publiée).

  • Dédoublonnage sur ISBN :

    Si oui, permet de dédoublonner les notices sur l'ISBN. Les notices avec un ISBN déjà existant dans le catalogue seront ignorées.

    Si plusieurs notices ont le même ISBN dans le fichier, seule la première sera importée.

    Ignorer les ISSN :

    Permet d'ignorer les notices ayant pour ISBN un ISSN. Cela est notamment le cas lorsque des bulletins de périodiques sont catalogués en tant que monographies.

  • Statut de notices importées :

    Ce statut permet de d'identifier rapidement les notices importées et d'en gérer la visibilité dans l'OPAC.

  • Générer les liens entre notices ?  :

    Si oui, crée les relations entre notices lors de l'import. Ce qui permet par exemple de générer le périodique et le bulletin desquels est issu un article.

  • Tenir compte des notices d'autorités :

    Si oui, importe des éléments dans les notices d'autorités (numéro d'autorité de l'auteur par exemple).

  • Origine par défaut des autorités si non précisé dans les notices :

    permet de définir l'origine des notices importées.

Transfert du fichier par préchargement⚓

Remarque

Si vous avez transféré votre fichier de notices directement sur le serveur, le logiciel n'offre pas la possibilité de choisir un fichier à télécharger, le reste du processus est identique.

Références

  1. UNIMARC

    Basé sur le format MARC (MAchine-Readable Cataloging), UNIMARC en est une variante créée par l'IFLA en 1977.

    Ce format est le format officiel d'échange de données bibliographiques en France.

  2. PHP

    PHP (sigle de PHP: Hypertext Preprocessor), est un langage de programmation libre, notamment utilisé pour la production de pages Web dynamiques.
