Définition de quotas ⚓

Les quotas permettent donc notamment de définir notamment les conditions de prêt et de réservation des exemplaires par les lecteurs.

2 types de quotas sont possibles :

  • des quotas avec vérification :

    Ces quotas vérifient les conditions qui s'appliquent pour un ensemble d'exemplaires et pour un lecteur, et comportent donc une valeur maximum.

  • des quotas sans vérification :

    Ces quotas retournent simplement les conditions qui s'appliquent à un exemplaire pour un lecteur.

Exemple

Les quotas sur le nombre de documents sont des quotas avec vérification. Un lecteur peut emprunter un certain nombre d'exemplaires.

Par contre les quotas sur la durée d'emprunt ou sur le nombre de prolongations sont des quotas sans vérification, car la durée d'emprunt et le nombre de prolongations ne concernent qu'un seul exemplaire.

Les quotas sont basés sur une liste pré-établie de caractéristiques des exemplaires et/ou des lecteurs :

  • catégories de lecteurs

  • supports

  • catégories de lecteurs et supports

  • lecteurs individuels

  • lecteurs individuels et supports

 

Pour chacun des éléments de cette liste, et pour chacune des valeurs possibles de ces éléments, un quota peut être défini.

Une valeur par défaut permet de préciser le quota qui s'applique si rien n'a été précisé dans les quotas par caractéristiques.

Dans le cas des quotas avec vérification, une valeur maximum permet de limiter l'ensemble des quotas définis par caractéristiques.

Cette valeur peut être ignorée si un quota par Catégories de lecteurs est précisé.

Exemple

Quotas par supports définis :

  • Support = "Livres"

    => Nombre de documents empruntables = 3

  • Support = "DVD"

    => Nombre de documents empruntables = 1

Valeur par défaut de nombre de documents si ce n'est pas précisé = 2

Dans ce cas, si l'on ajoute un support "CD", le nombre de documents empruntables est de 2 (jusqu'à ce que l'on définisse un quota pour le support "CD").

Le total ne doit jamais dépasser = 3

Il est donc possible d'emprunter 3 "Livres", ou 2 "Livres" et 1 "DVD", mais pas 3 "Livres" et 1 "DVD", car dans ce dernier cas, le nombre de documents maximum empruntables est dépassé.

Gestion des conflits⚓

Afin d'arbitrer le résultat si plusieurs quotas sont applicables, un système de résolution de conflits entre quotas est également défini, qui permet de choisir le quota applicable :

Exemple

Quotas par supports définis :

  • Support = "Livres"

    => Nombre de documents empruntables = 3

  • Support = "DVD"

    => Nombre de documents empruntables = 1

Quotas par Catégories de lecteurs définis :

  • Catégorie = "Collectivités"

    => Nombre de documents empruntables = 20

Valeur par défaut de nombre de documents si ce n'est pas précisé = 2

Le total ne doit jamais dépasser = 3

La case sauf si il y a un quota par Catégories de lecteurs précisé est cochée.

Si l'on choisit :

  • Prendre le plus grand, il est possible d'emprunter 20 "Livres".

  • Prendre le plus petit, il n'est possible d'emprunter que 3 "Livres".

  • Prendre la valeur par défaut, il n'est possible d'emprunter que 2 "Livres".

  • Regarder dans l'ordre, le premier quota rencontré sera celui choisi.

Forçage⚓

Remarque

Cette option n'est disponible que pour les quotas avec vérification.

Les quotas peuvent être incontournables, ou forcés.

Si l'on choisit d'autoriser le forçage des quotas en règle générale, les quotas par caractéristiques pourront être définis comme incontournables.

Inversement, les quotas par caractéristiques pourront être forcés.

Exemple

Dans le cadre du prêt, si l'on autorise un forçage, un message avertira du dépassement du nombre de prêts autorisés et le prêt pourra être forcé.

Dans le cas contraire, seul le message sera émis, le prêt ne pourra pas se faire.
