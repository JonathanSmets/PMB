Gérer les pages de son portail

L'intérêt d'un CMS[1] réside dans le fait que le contenu éditorial (rubriques, articles) et la conception du site sont séparés. Ainsi, au lieu de créer une page par information, le portail PMB permet de créer une page qui servira de modèle pour, par exemple, l'ensemble des articles du portail.

La création d'une pageâš“

Ce menu permet de créer une page. Le paramétrage des informations qui seront affichées sur cette page est abordé dans le menu Portail > Construction : Construire.

Pour créer une page, cliquez sur Ajouter une page.

Plusieurs informations peuvent être remplies :

  • Nom de la page : titre de la page (obligatoire),

  • Contenu de la page : description de la page,

  • Variables d'environnement : les variables d'environnement permettent de paramétrer les informations affichées dans la page. Une page n'a pas l'obligation de comporter une variable d'environnement. Cependant pour gérer l'affichage dynamique d'éléments du contenu éditorial, il faut pouvoir indiquer quels sont les identifiants qui seront transmis dans l'URL.

    • Le premier champ correspond à l'argument qui sera transmis dans l'URL, seuls les caractères alphanumériques non accentués et le "_" sont tolérés.

    • Le deuxième champ sert à repérer facilement la variable lors du paramétrage des modules, le format de ce champ est libre.

Enregistrez la page pour valider sa création.

Truc & astuce

Pour savoir si la page que vous créez a besoin d'au moins une variable d'environnement, posez-vous la question suivante :

  • La page peut-elle être utilisée de manière générique pour afficher du contenu éditorial différent ?

À tout moment, vous pouvez modifier ou supprimer la page en cliquant sur son nom.

Attention

Vous pouvez supprimer une page alors même qu'elle contient des cadres et qu'elle est utilisée sur votre portail. Ainsi, prenez garde à ne pas supprimer inopinément une de vos pages sous peine de casser la structure de votre portail.
