Mettre en place une veille

Accéder au module de veille⚓

La seule condition d'accès au module de veille est d'avoir accès à l'onglet DSI de PMB. Aucun autre paramétrage n'est nécessaire, l'outil de veille est activé par défaut.

Pour y accéder, aller dans l'onglet D.S.I > Veilles > Définition des veilles.

Créer un classement⚓

Un classement permet de regrouper une ou plusieurs veilles, sur une thématique par exemple. Pour le créer, cliquez sur le bouton Nouveau Classement.

Choisissez un classement parent et un nom pour le classement. Le contenu d'un classement peut être diffusé sur le portail.

Créer une veille⚓

Une veille doit être dans un classement. Pour la créer, cliquez sur le bouton Nouvelle veille.

Elle doit également avoir un nom, une durée de TTL (Time To Live). Une description ainsi qu'un logo peuvent être ajoutés.

Sous la liste des Options RSS, vous pouvez inscrire les informations permettant de sourcer sur le flux généré par cette veille.

Il est possible de paramétrer l'accès à la veille selon les différents utilisateurs.

Il est également possible de paramétrer des données par défaut dans le cas où un item de la veille serait transformé en notice, en article ou en rubrique du site.

Un dernier champ permet de filtrer les futurs items pour que n'apparaissent que ceux ayant les termes inscrit dans ce champ.

Il est à noter que ce champ est également présent au niveau des sources.

Vous obtenez alors la structure suivante :

Ajouter une source⚓

Positionnez-vous sur la veille, cliquez sur le bouton Ajouter une nouvelle source et choisissez le type de source à ajouter.

Articles de la base : permet de définir une veille sur les articles d'une ou plusieurs rubriques du portail

Rubriques de la base : permet de définir une veille sur une ou plusieurs rubriques du portail

Notices de la base : permet de créer un flux RSS sur les notices contenues dans un ou plusieurs paniers

Notices RSS de la base : permet de créer un flux RSS sur le contenu des flux RSS mis en "Lien (ressources électroniques)" de notices de la base contenues dans un panier.

Flux RSS : permet d'agréger un flux RSS externe

Notices dans les entrepôts : permet de récupérer les notices stockées dans un entrepôt paramétré via un connecteur.

Surveillance de site : nécessite une extension permettant de cibler les chemins XPath sur un site web, permet de surveiller les nouveaux contenus sur une page ou un site.

Paramétrer la source⚓

Nommer votre source

Renseigner un TTL (Time To Live) pour paramétrer le temps de rafraîchissement de la source (en heure)

Marquer les nouveaux éléments comme intéressants par défaut : Permet de publier l’intégralité du flux sur le portail en sélectionnant automatiquement tous les items

Éliminer la mise en forme HTML des nouveaux éléments : permet de récupérer l'information "brute" et d'appliquer la mise en forme du portail

Filtrer les nouveaux éléments avec une expression booléenne : inscrire ici une recherche permettant de filtrer les items récupérés au niveau de la source. Si jamais un filtre avait déjà été appliqué au niveau de la veille, celui-ci s'y ajoutera.

Récupérer les données via : sélectionner la source souhaitée. La présentation de ce paramètre diffère selon le type de source sélectionné.

Résultat final⚓

Vous obtenez ainsi un tableau récapitulant les différents items des sources agrégées, quelle que soit la nature de la source.

La rubrique "Items" vous permet donc d’accéder aux items issus de la veille.

La rubrique "Sources" vous permet de visualiser et, au besoin, de modifier les sources de la veille.
